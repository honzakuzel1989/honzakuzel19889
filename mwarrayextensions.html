MWArray.Extensions

Everyone who have ever worked with MWArray class or with inherited classes knows that API of these classes is not very friendly. That was the main reason because I created a friendly, powerfull, fast, extensible and well-tested set of MWArray class extensions.

The set contains only a few self-explaining methods: ToSingleItem, ToArray, ToEnumerableOfArrays, GetField, GetFieldAsSingleItem, GetFieldAsArray, CreateArray, CreateItem. As you can see below, the methods significantly simplify work with MWArray based classes and improve code writability and readability.

If you want to try these extensions for NON-COMMERCIAL use you can install MWArray.Extensions.R2017a NuGet package from official NuGet packages source.

Packages

The MWArray.Extensions library can be build with several versions of MWArray library for several types of operating system (it depends on MATLAB Runtime Version). The package version of the MWArray is 2.17.0.0 from MATLAB Runtime codename R2017a version 9.2. Required operating system for the package is Windows. You must follow these restrictions as same as MATLAB, MATLAB Runtime and MATLAB Compiler restrictions (in readme.txt) otherwise extensions probably will not work!

ToSingleItem

This method serves for convert some MWArray based classes (specifically MWNumericArray and MWLogicalArray) to single item of desired type. Because the method (as well as other methods) internally uses Convert.ChangeType you can choose any convertible output type (string for example). The output type is defined like a generic parameter of the method.

If the source array contains more than one item the method returns the first one (at coordinate [0,0]). If the source array is empty the method throws ArgumentException. As you can see in example (with extension vs. without extension) the method simplify code and make additional conversion implicitly.

For more information you can explore tests of the method.

ToArray

This method is usually use for convert some MWArray based classes (specifically MWNumericArray and MWLogicalArray) to array of items of desired type. Because the method also internally uses Convert.ChangeType you can choose any convertible output type (string for example). The output type is defined like a generic parameter of the method.

If the source array contains only one item the method returns the one item array. If the source array is empty the method returns empty array of return type. As you can see in example (with extension vs. without extension) the method simplify code and make additional conversion implicitly.

For more information you can explore tests of the method and benchmark (1 2) of the method.

ToEnumerableOfArrays

This method convert some double-dimensional MWArray based classes (specifically MWNumericArray and MWLogicalArray) to enumerable of arrays of desired type. Because the method inside ArrayHelper also internally uses Convert.ChangeType you can choose any convertible output type (string for example). The output type is defined like a generic parameter of the method.

If the source array contains only one item the method returns the enumerable with one array with one item. If the source array is empty the method returns empty enumerable of array of return type. As you can see in example (with extension vs. without extension) the method simplify code and make additional conversion implicitly.

For more information you can explore tests of the method.
